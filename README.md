
# Dispatch QR Platform (Liquid Glass) — FastAPI + Jinja + Tailwind

A premium, browser-based dispatch sharing platform inspired by iOS "Liquid Glass" (translucent layered surfaces, soft depth, and gentle motion).

## What this platform includes
- Depot dashboard (create, edit, search/filter, generate QR, share via WhatsApp/email)
- Public dispatch page (opened via QR/shared link)
- Public page shows ONLY safe fields:
  - vehicle number
  - dispatch date
  - station/location (district + GPS pin)
  - ICN number (dispatch-level cross-check ID)
  - driver name
  - driver mobile
  - number of packages
  - total weight (kg)
  - likely arrival date
- Seed/demo data on first run (safe fields only on public pages)

## Local run (Windows PowerShell)
```bash
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python -m uvicorn app:app --reload --port 8000
```

## One-click start (recommended)
On Windows, run `start.bat` from the project root. It sets up the venv, installs requirements, starts the backend, and opens the dashboard in a browser.

## GPS seeding (district lat/lng)
The app uses a `locations` table (SQLite) for the public GPS pin.

Run once after installing dependencies:
```bash
python seed_locations.py
```

## PIN-code GPS seeding
For PIN-code based GPS + approx distance, run:
```bash
python seed_pincodes.py
```

## PIN autocomplete + depot distance filters
- `New dispatch` / `Edit dispatch` support PIN auto-resolve (place/state) using the backend endpoint `/pincode/resolve/{pin}`.
- Dashboard supports `Distance range` filter from the depot.

## Depot distance reference
The "approx distance from depot" is computed using depot coordinates inside `app.py` (`DEPOT_LAT`, `DEPOT_LNG`). It is currently set to **Banar, Rajasthan**; adjust those constants if your depot location differs.

## Key URLs
- Depot dashboard: http://127.0.0.1:8000/
- Create dispatch: http://127.0.0.1:8000/admin/new
- Edit dispatch: http://127.0.0.1:8000/admin/edit/{token}
- Share + QR console: http://127.0.0.1:8000/admin/share/{token}
- Public dispatch page: http://127.0.0.1:8000/dispatch/{token}
- QR image endpoint (used by the UI popup): http://127.0.0.1:8000/qr/{token}.png

## Create a new dispatch
1. Open the depot dashboard.
2. Click `New dispatch`.
3. Fill in the dispatch fields (include `ICN number` + `Total weight (kg)`) and submit.
4. You will be redirected to the Share + QR console for the new token.

## Generate & share a QR
On the Share + QR console:
- `View QR` (popup layer; no direct image download button)
- `Copy public link`
- `WhatsApp share`
- `Email share`

The QR always points to the public dispatch page URL.

## Render deployment (recommended path)

This repo now includes `render.yaml` for one-click Render setup.

### Why this setup
- Runs with `gunicorn` + Uvicorn worker
- Includes health check endpoint: `/healthz`
- Uses persistent disk (`/var/data`) so SQLite data survives restarts/deploys

### Steps on Render
1. Push this project to GitHub.
2. In Render: **New** -> **Blueprint** -> select your repo.
3. Render reads `render.yaml` and creates service settings automatically.
4. In service env vars, set:
   - `PUBLIC_BASE_URL` = your Render URL (or custom domain), e.g. `https://your-app.onrender.com`
   - `ADMIN_PASSWORD` = your password (or keep current if desired)
5. Deploy.

### Important env vars
- `PUBLIC_BASE_URL` (required for correct QR links)
- `SESSION_SECRET` (auto-generated by Render blueprint)
- `DISPATCH_DB_PATH=/var/data/dispatch.db` (already configured in `render.yaml`)
- `ADMIN_USERNAME` / `ADMIN_PASSWORD`

### Notes
- Public users can open only QR/public dispatch pages.
- Dashboard/admin routes require login.
- If you add a custom domain, update `PUBLIC_BASE_URL` to that domain and redeploy.
